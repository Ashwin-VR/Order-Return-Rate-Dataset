# Detailed EDA Plan for Synthetic E-commerce Return Dataset

This plan is tailored to the deterministic dataset generated by `final_moments/data_generation.py` and evaluated by `final_moments/task/eda_and_metrics.py`.

Main objectives:
- Validate dataset quality and rule consistency.
- Explain drivers of `return_probability` and `is_returned`.
- Detect leakage and realism gaps before modeling.
- Produce business-ready insights and modeling guidance.

## 0. Inputs and Expected Schema

Preferred analysis file:
- `final_combined_data.csv`

If it does not exist, build it from:
- `orders.csv`
- `customers.csv`
- `products.csv`

Expected columns:
- Identifiers: `order_id`, `customer_id`, `product_id`
- Time: `order_date`
- Operational: `delivery_delay`, `discount_pct`, `is_holiday_season`
- Customer: `persona`, `base_return_rate`, `customer_tenure_days`, `avg_order_value`
- Product: `category`, `price`, `rating`
- Targets: `return_probability`, `is_returned`

## 1. Dataset Overview

Checklist:
- Report shape (rows, columns).
- Report dtypes and split into numerical vs categorical columns.
- Check duplicate rows.
- Check uniqueness of `order_id`.
- Report unique counts for `customer_id` and `product_id`.

Pass criteria:
- No duplicate `order_id`.
- Row count near configured generator size (default 10000).

## 2. Data Quality and Constraint Validation

Mandatory checks:
- Missing values: count and percentage per column.
- Domain checks:
   - `return_probability` in [0, 1]
   - `is_returned` in {0, 1}
   - `delivery_delay` in {0, 1, 2, 3, 5, 10}
   - `is_holiday_season` in {0, 1}
   - `order_date` within generated year window
- Category-specific bounds:
   - Electronics ratings in [3.0, 5.0]
   - Apparel and Footwear ratings in [2.0, 4.5]
   - Home and Beauty ratings in [3.5, 4.9]
- Persona-specific discount policy checks:
   - Bargain Hunter discounts in {20, 30, 50, 70}
   - Others in {0, 5, 10, 20}

Outlier review:
- Use IQR summary by numeric column.
- Do not auto-remove outliers; mark as intentional or suspicious.

## 3. Target Variable Analysis

### 3.1 Regression target: `return_probability`

Required outputs:
- Histogram + KDE.
- Summary statistics: mean, median, std, skewness.
- Binned distribution counts:
   - [0.0, 0.2)
   - [0.2, 0.5)
   - [0.5, 0.8)
   - [0.8, 1.0]

Interpretation questions:
- Is distribution heavily left/right skewed?
- Is there a clear risky zone around threshold 0.5?

### 3.2 Classification target: `is_returned`

Required outputs:
- Class count and class ratio.
- Imbalance ratio.
- Consistency test:
   - `is_returned == (return_probability > 0.5)`
   - Report mismatch count (should be 0 for current generator).

## 4. Categorical Analysis

For each of `persona`, `category`, `is_holiday_season`:
- Count distribution.
- Mean `return_probability`.
- Return rate percentage (`mean(is_returned) * 100`).
- Visuals: countplot and barplot for mean target.

Business interpretation focus:
- Highest-risk customer segment.
- Highest-risk product segment.
- Seasonal sensitivity.

## 5. Numerical Analysis

For each numeric feature (`delivery_delay`, `discount_pct`, `rating`, `price`, `customer_tenure_days`, `avg_order_value`, `base_return_rate`):
- Distribution plot (histogram or KDE).
- Boxplot by `is_returned`.
- Scatter vs `return_probability` with trendline when meaningful.

Special validation:
- Check non-linear tenure relationship with `return_probability` (log-decay behavior).
- Check rating effect below 3.0.

## 6. Interaction and Rule Verification

Test the deterministic rules directly using grouped comparisons.

Rule A: Delay impact by category
- Compare high delay vs low delay for Apparel/Footwear.
- Compare high delay vs low delay for Electronics.
- Expected direction: stronger penalty for Apparel/Footwear than Electronics.

Rule B: High discount on Apparel
- Compare Apparel with `discount_pct >= 50` vs `< 50`.
- Expected direction: higher return probability in high-discount group.

Rule C: Holiday impulse effect
- Compare Impulse Buyer in holiday vs non-holiday orders.
- Expected direction: holiday group has higher return probability.

Rule D: Base signal contribution
- Plot `base_return_rate` vs `return_probability`.
- Confirm positive relationship with controlled spread due to interactions and noise.

## 7. Correlation and Multivariate Analysis

Required analyses:
- Correlation matrix heatmap for numeric columns.
- Correlation ranking with `return_probability`.
- Pairplot on sampled rows if needed for speed.

Important caveat:
- Correlation may understate non-linear and threshold effects.

## 8. Leakage, Bias, and Realism Checks

Leakage checks:
- `base_return_rate` is generated upstream of target, so treat as high-risk leakage for predictive modeling.
- Exclude identifiers and direct label proxies from training features.

Bias checks:
- Identify if one variable dominates too strongly.
- Compare feature importance intuition vs model importances.

Realism checks:
- Determine whether deterministic rules are too sharp (overly easy prediction).
- Note if additional stochastic variation is required for production-like behavior.

## 9. Feature Engineering Recommendations

Candidate features:
- `high_delay_flag = delivery_delay > 3`
- `rating_under_3_flag = rating < 3.0`
- `delay_x_apparel = (category in {Apparel, Footwear}) * delivery_delay`
- `holiday_x_impulse = is_holiday_season * (persona == Impulse Buyer)`
- `log_tenure_days = log1p(customer_tenure_days)`

Modeling split guidance:
- Regression task: predict `return_probability`.
- Classification task: predict `is_returned`.
- Never include target or post-target fields as features.

## 10. Metric Reporting Block

Report at minimum:
- Regression: $R^2$, RMSE, MAE
- Classification: accuracy, precision, recall, F1, ROC-AUC

Recommended baseline thresholds for this deterministic dataset:
- Regression: $R^2 > 0.85$, RMSE < 0.08, MAE < 0.06
- Classification: F1 > 0.85

If metrics are too perfect:
- Flag potential over-determinism or leakage.

## 11. Required Visual Set

Mandatory chart types:
- Histograms
- Boxplots
- Countplots
- Heatmap
- Scatterplots

Formatting rules:
- Clear title per chart.
- Axis labels and units where relevant.
- Consistent color palette across sections.

## 12. Final Deliverables

Produce all of the following:
1. One EDA report with key findings and tables.
2. One metrics summary block including $R^2$, RMSE, MAE.
3. One risk section with leakage and realism limitations.
4. One action section with improvements for data realism and model robustness.

Executive summary must answer:
- Which customers are highest risk?
- Which categories are highest risk?
- How much do delay, discount, and rating contribute?
